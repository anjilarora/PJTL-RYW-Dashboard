# Model card: xgboost_readiness_stage3_v2

## Summary

Binary classifier that predicts whether a Ride YourWay prospective market
is **Ready to launch** given the nine gate features. Output is a
calibrated probability plus a label at the 0.5 threshold.

## Intended use

- **Primary use**: decision-support inside the Ride YourWay readiness
  dashboard. Rendered as `p(Ready)` next to the nine-gate scorecard.
- **Secondary use**: driving the sensitivity sliders on
  [pages/index.vue](../../frontend/pages/index.vue) so analysts can
  explore what-if scenarios.
- **Not for**: regulated decisions, operational routing, pricing.

## Inputs (features)

Exactly the nine features enumerated in
[ml/feature-reference.md](feature-reference.md), in the order returned by
`engine.kpi_config.feature_order()`. The API schema is
`InferenceRequest` in
[code/backend/api/schemas.py](../../backend/api/schemas.py).

## Outputs

- `p_ready: float in [0, 1]` - calibrated probability of Ready.
- `label: "Ready" | "Not Ready"` - threshold at 0.5.

## Training data

- Source: `code/intermediates/inference_inputs/readiness_training_base.csv`.
- Provenance chain:
  - Generated by
    [code/scripts/generate_readiness_training_rows.py](../../scripts/generate_readiness_training_rows.py)
    (bulk + boundary + flip populations).
  - Labeled by
    [code/scripts/build_readiness_training_base.py](../../scripts/build_readiness_training_base.py)
    using the nine gate rules.
- Class balance: ~50/50 by construction because boundary/flip rows ensure
  both labels are well-represented near the decision surface.

## Limitations

- **No real-market training data yet.** The model is trained entirely on
  synthetic rules-labeled rows. It reproduces the gate rules with a
  smoothed boundary but does not encode any additional market insight
  beyond what the gates already capture.
- **Sparse acuity coverage.** The `high_acuity_share` feature has a
  modest range in the example workbook; extrapolation far above it is
  unvalidated.
- **Cost ceiling bias.** Because `cost_per_road_hour` is pinned at $50
  whenever the charter ceiling is active, the feature contributes less
  signal than the other eight in practice. The model still fits it for
  future real-market data where cost varies.

## Ethical considerations

The model does not touch rider PII, protected health information, or
protected-class attributes. All features are aggregated fleet-level
operating metrics. The decision it informs is commercial (launch or
not), not individual.

## How to reproduce

```bash
cd code
python scripts/build_phase1_canonical_base.py
python scripts/generate_readiness_training_rows.py
python scripts/build_readiness_training_base.py
python inference_engine/scripts/sync_inputs_from_phase1.py
jupyter nbconvert --to notebook --execute inference_engine scripts
```

The stage-3 notebook is authoritative. The earlier script
`train_readiness_model_from_inputs.py` produces the `inputs_v1` artifact
that CI uses for regression checks.
