## 2) What we built in the app

The app is a Nuxt SPA + FastAPI backend with two main experiences:

- **Q1 baseline dashboard** (`/dashboard`):
  - Reads baseline viability from `GET /api/v1/viability/baseline`
  - Shows readiness, gates, intake summary, operational deep dive, execution queue
  - Operational deep-dive panels are backed by `GET /api/v1/dashboard/*` endpoints

- **Prospective market evaluator** (`/market`):
  - Uploads a prospective intake `.xlsx`
  - Backend job runs intake extraction -> contract mapping -> baseline merge -> viability + ML inference
  - Returns readiness verdict, gate breakdown, ML probability, and mapped contract table

- **Model sandbox** (`/` home):
  - Nine slider controls for gate features
  - Calls `/api/v1/inference/predict` for instant what-if scoring

## 3) What the tabs represent

## Top navigation
- `Home`: nine-gate model sandbox (interactive sliders)
- `Dashboard`: frozen Q1 operational/baseline story
- `Market Readiness`: upload and score a new prospective workbook
- `Settings`: app-level settings page

## Dashboard section tabs
- `Operating snapshot`: KPI summary and baseline highlights
- `Gate readiness`: gate carousel + selected gate details
- `Intake & programs`: intake mix summary panel
- `Operational deep dive`: D1..D10 operational analytics panels
- `Execution queue`: risk/action queue panel

- D4 OTP matrix

- D7 Cancellations
